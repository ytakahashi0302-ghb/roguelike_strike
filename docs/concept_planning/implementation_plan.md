# 実装計画

プロジェクト「Rogue Strike」の初期計画とコンセプト設計を行います。

## 概要
「ローグライク×モンスト×ブロック崩し×インベーダー」というコンセプトのゲームを開発する。
現在はコンセプト計画段階であり、ゲームのルール、キャラクター特性、敵の特性などの仕様策定を行っている。

## テスト方針
- **ロジックテスト**: 射出角度と反射の物理演算、敵の進行ロジックなどの単体テスト。
- **プレイテスト**: 実際の射出感、敵が降りてくるスピード、難易度曲線を実機（ブラウザ等）上で目視・手動検証する。

## 実装ステップ
1. **仕様の確定**: 本チャットのやり取りを通じて仕様書（`specification.md`）を完成させる。
2. **技術スタックの決定**: Vite + Vanilla TypeScript。Canvas APIを用いた軽量な2D描画。GitHub Pagesへのデプロイを前提とした静的サイト構成。
3. **プロトタイプ構築**: 
   - Canvasの初期化とゲームループの実装
   - 自キャラの配置と、引っ張り操作（射出方向の決定）の実装
   - 壁でのバウンドと摩擦による停止ロジック
   - 簡易的な敵ブロックの配置と当たり判定
   - キャラ停止時に敵が1段下がるターン進行の実装

## User Review Required
> [!IMPORTANT]
> ゲームコンセプト、キャラクター特性、敵の特性について、ユーザーのレビューと合意が必要です。`specification.md` をご確認ください。

## Verification Plan
### Automated Tests
- 現状は仕様策定フェーズのため、コードレベルのテストはなし。プロトタイプ作成時に単体テストを導入予定。

### Manual Verification
- 仕様書の内容がユーザーの意図したコンセプトと合致しているかを確認。
